LiteScope: Reveal The Security Paradox of Android Lite Apps in Resource-Constrained Ecosystems

Anonymous Authors

Abstract

Android Lite Apps, streamlined versions of full-featured applications, gain significant popularity, particularly among low-end devices and bandwidth-constrained networks. However, Lite Apps receive widespread bug complaints in Google Play, even for world-famous apps like Facebook Lite. This contradicts the common belief that fewer features improve robustness. Unfortunately, understanding this phenomenon remains limited and challenging. To be specific, we identify three core challenges: (C1) asymmetric version alignment between Lite and Full Apps, (C2) lack of a comprehensive large-scale crash dataset, and (C3) semantic correspondence complexity stemming from code divergence and obfuscation. To tackle these challenges, we present LiteScope, a prototype that systematically aligns Lite-Full versions using semantic and structural heuristics, generates a Lite-specific crash dataset via differential bug-triggering, and performs root cause analysis through semantic comparison. Through the evaluation of 536 Lite-Full Apps pairs, we trigger 179 Lite-specific crashes, from which we distill five critical findings. Based on these insights, we formulate actionable recommendations for developers to enhance the stability and security of Lite Apps.
