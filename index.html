LiteScope: Reveal The Security Paradox of Android Lite Apps in Resource-Constrained Ecosystems

Abstract

Android Lite Apps, streamlined versions of full-featured applications, gain significant popularity, particularly among low-end devices and bandwidth-constrained networks. However, Lite Apps on Google Play receive widespread bug complaints, even for world-famous apps like Facebook Lite. This phenomenon contradicts the common belief that fewer features improve robustness. Unfortunately, understanding this phenomenon remains limited and challenging. To be specific, we identify three core challenges: (C1) asymmetric version alignment between Lite and Full Apps, (C2) lack of a comprehensive large-scale crash dataset, and (C3) semantic correspondence complexity stemming from code divergence and obfuscation. To tackle these challenges, we present LiteScope, a prototype that systematically aligns Lite–Full versions using semantic and structural heuristics, generates a Lite-specific crash dataset via differential bug-triggering, and performs root cause analysis through semantic comparison. Through the evaluation of numerous Lite–Full Apps pairs, we triggered a significant number of Lite-specific crashes, from which we distill five critical findings.

Datasets

Crashes information, based on the Lite-specific crashes detailed in the paper, is available in the `crashes.csv` file below. Our research and testing efforts are ongoing, and the number of identified crashes has since grown. We are committed to providing comprehensive information to developers.

For security reasons and to protect app developers, we have eliminated the exact versions of the apps in the publicly shared information.

Field Descriptions

crashes.csv (Summary File)

  • id: Unique identifier for the crash instance.
  • type: The type or category of the crash (e.g., NullPointerException).
  • long_msg: A detailed message or description associated with the crash.
  • details: Further specific details or stack trace information.
  • file: The source file where the crash occurred, if available.

ids.csv

  • lite_id: The package identifier for the Lite version of the app.
  • lite_name: The display name of the Lite app.
  • full_id: The package identifier for the Full version of the app.
  • full_name: The display name of the Full app.
